Design Polish Priorities

Priority 1: Main Surfaces

  • Today: 정보량이 가장 많고, source strip / sync banner / timeline hierarchy가 앱 인상을 좌우합니다.
  • Persons: card density, pending badge, snippet spacing, offline badge tone 정리가 효과가 큽니다.
  • Person Detail: pending/completed/history의 시각적 대비와 section rhythm을 강화할 여지가 큽니다.

Priority 2: High-Frequency Task Surfaces

  • Commitment Management / Detail / Edit / Create: 상태칩, destructive action, due 표현의 hierarchy를 통일하면 제품 완성도가 크게 올라갑니다.
  • Sources / Source Detail: source owner가 정리된 만큼, status/action hierarchy를 더 product-like 하게 정돈할 수 있습니다.

Priority 3: Onboarding Funnel

  • Login + PIPA + provider connect screens: 기능은 닫혔으므로 이제 typography, copy density, CTA emphasis를 다듬기 좋습니다.
  • Cold Sync: progress 피드백과 “지금은 건너뛰기”의 위계가 제품 신뢰감에 영향을 줍니다.

Immediate Low-Cost Wins

  • 실제 브랜드 폰트 적용
  • section spacing / vertical rhythm pass
  • status chip / badge / secondary text contrast 조정
  • sheet 등장/닫힘과 section expand의 motion grammar 보강
