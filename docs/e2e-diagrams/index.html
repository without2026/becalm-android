BeCalm Android — E2E Wiring Diagrams

User behavior 가 코드 위에서 어떻게 흐르는지, 함수 · 라인 번호까지 추적한 sequence diagram 모음

목적

Wave 기반 구현이 진행되면서 CTO 가 코드를 직접 읽지 않고도 UI → VM → Repo → Remote → Persist 체인을 눈으로 따라갈 수 있도록, 각 user behavior 를 mermaid sequence diagram + 파일:라인 참조 로 정리합니다.

  • 모든 줄번호는 현재 feat/wave-1-impl 브랜치 기준 실제 코드 위치입니다.
  • UI 에 아직 배선되지 않은 계약상 경로는 미배선 표시.
  • 성공/실패 두 흐름을 항상 분리. 실패 시 어떤 세션/토큰/상태가 오염되지 않는지도 명시.

참고 문서

문서용도
docs/e2e-verification/*.md 모듈별 단계 | 파일 | 심볼 3-column table. 이 HTML 의 1차 소스.
docs/plans/_wave-plan.md 9-wave 구현 순서. 각 wave 가 건드리는 파일 예측.
.spec/*.spec.yml behavior 계약 (MUST-language). AUTH-001 등 spec ID 는 여기서 유래.

규약

  • 성공 경로는 "observer-driven state transition" 을 따름 — VM 이 직접 SignedIn 등으로 전이하지 않고, Repository 의 Flow 를 collect 해서 상태를 결정.
  • 실패 경로는 네트워크/권한/SDK 예외를 BecalmError 로 매핑 후 UI 에 safeMessage 로 노출.
  • 미배선 은 VM/Repo/Client 가 정의되어 있으나 UI 에서 호출 지점이 없음. 구현 시 이 표시부터 제거.
  • invariant 은 spec 이 요구하는 불변식 — 코드 리뷰 시 반드시 확인.

사용법

이 폴더의 HTML 파일을 브라우저에서 직접 열면 됩니다. 빌드 불필요.

# macOS
open becalm-android/docs/e2e-diagrams/auth.html

# Linux
xdg-open becalm-android/docs/e2e-diagrams/auth.html

# Windows
start becalm-android\docs\e2e-diagrams\auth.html
Mermaid 는 CDN 에서 로드됩니다 (unpkg.com/mermaid). 오프라인 환경이면 mermaid.min.js 를 이 폴더에 다운로드하고 <script src> 를 로컬 경로로 바꾸세요.
