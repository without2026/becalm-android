01 · Auth — Login Flow Wiring

Email / Google OAuth 로그인의 성공·실패 경로. feat/wave-1-impl 기준 · spec AUTH-001 / 002 / 003

중요 — Google sign-in UI 미배선: ui/auth/LoginScreen.kt:177-181 의 R.string.login_google_cta 는 clickable 버튼이 아니라 Text 라벨입니다. AuthViewModel.onGoogleSignIn 을 호출하는 composable 이 현재 프로젝트에 없음 (grep onGoogleSignIn 결과 = 정의만 존재). 아래 Google 다이어그램은 "계약상 의도된 경로" 이며 실제로는 Wave 6 의 ui-auth-google-signin-wiring 이 구현 예정.

공통 관찰 · invariant

  • 상태 전이의 단일 출처: onEmailSignIn / onGoogleSignIn 은 성공 시 SignedIn 으로 직접 전이하지 않음. sessionFlow (AuthRepository.kt:143) → observeAuthState() (L280) → onObserveSession collector (AuthViewModel.kt:165-183) 가 담당. VM 주석 L90-92, L135-139 에 "uiState will update via onObserveSession" 로 invariant 명시.
  • 실패 시 비오염: signInWithEmail / signInWithGoogle 의 .onSuccess { ... } 체인 (L150, L160) 은 실패 시 실행되지 않음. setCurrentUserId / sessionFlow.value / primeCache 모두 건너뜀.
  • 토큰 저장 invariant: 모든 토큰은 EncryptedSharedPreferences (Android Keystore 기반) 에만 저장. SupabaseClientFactory 에서 autoLoadFromStorage = false / alwaysAutoRefresh = false 로 SDK 의 plaintext prefs 사용 차단 (SupabaseAuthClient.kt:98-107 주석 참조).
  • 토큰 캐시 프라이밍: 성공 시 tokenProvider.primeCache() 호출 (L155, L163) 로 다음 hot-path 요청이 디스크 읽기 없이 Bearer 헤더 구성 가능. Round 6A.4 성능 개선.
  • 에러 메시지 sanitize: UI 는 raw exception 을 노출 금지. BecalmError.safeMessage 통과 필수 (AuthViewModel.kt:104, 125).
